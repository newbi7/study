BM25(Best Match 25)
Bag-of-words 개념을 사용하여 각각의 문서에 얼마나 자주 등장하는지를 평가함
score(D,Q) = ∑((tf(w,D) * idf(w) * (k + 1)) / (tf(w,D) + k * (1 - b + b * (|D| / avgdl))) * qtf(w,Q))
D는 문서(Document)를 나타냅니다.
Q는 쿼리(Query)를 나타냅니다.
w는 D와 Q에서 등장하는 단어(Term)를 나타냅니다.
tf(w,D)는 D에서 w의 등장 빈도수(Term Frequency)를 나타냅니다.
idf(w)는 전체 문서에서 w가 등장하는 문서 빈도의 역수(Inverse Document Frequency)를 나타냅니다.
k와 b는 상수값으로, k는 문서 길이에 대한 가중치를, b는 문서의 짧은 단락에 대한 가중치를 조절합니다.
|D|는 문서 D의 길이를 나타냅니다.
avgdl은 평균 문서 길이를 나타냅니다.
qtf(w,Q)는 쿼리 Q에서 w의 등장 빈도수를 나타냅니다.
BM25는 이러한 수식을 이용하여 문서와 쿼리 간의 유사도를 계산하고, 가장 관련성이 높은 문서를 상위로 랭킹하는 데 사용됩니다.

파라미터는 함수나 메서드 정의 시 괄호 안에 선언되는 변수를 말합니다. 이 변수들은 함수나 메서드에서 사용될 값들을 받아들이는 역할을 합니다. 파라미터는 함수나 메서드를 호출할 때, 해당 함수나 메서드에 필요한 입력값을 전달하기 위해 사용됩니다.

예를 들어, 다음과 같은 자바 메서드가 있다면:

java
Copy code
public void greet(String name) {
    System.out.println("Hello, " + name + "!");
}
여기서 name은 파라미터입니다. greet 메서드가 호출될 때, 파라미터 name은 호출한 곳에서 전달된 값으로 대체됩니다.

아규먼트는 함수나 메서드를 호출할 때 전달되는 값입니다. 즉, 파라미터와 대응되는 실제 값이라고 할 수 있습니다. 함수나 메서드를 호출할 때, 괄호 안에 아규먼트를 전달함으로써 해당 함수나 메서드에 필요한 입력값을 제공합니다.
